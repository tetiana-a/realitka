01 Context

Бізнес, який живе у поштовій скриньці

RealityPraha — невелика realitní kancelář у Празі: 6 людей (3 makléři, юрист, office-менеджер і директор), 15–30 звернень на день через info@realitypraha.cz та форму на сайті.

Поточний процес (до автоматизації)

  • Усі листи падають у спільну скриньку info@realitypraha.cz
  • Makléři вручну читають і сортують пошту 2–3 рази на день
  • Пріоритет визначається «на око» — залежно від настрою і завантаженості
  • Юридичні питання та скарги змішані зі стандартними дотазами і рекламою

Команда та категорії

  • Property Inquiry / Viewing Request → Jan Novák
  • Pricing → Petra Horáková
  • Complaint → Martin Šimek (office manažer)
  • Legal Question → Eva Čermáková (právník)
  • Spam → ignorovat
02 Problem & Solution

VIP-клієнт чекає в одній черзі зі спамом

Найдорожча дірка в клієнтському сервісі — час між тим, як прийшов лист, і тим, як хтось узяв його в роботу.

Що не працювало

  • VIP-запит на перегляд квартири за 8,5 млн Kč губився серед реклами та "Як підготуватися до прохідки"
  • Скарга з погрозою судовим позовом могла чекати відповіді кілька днів
  • Жодного SLA — "відповімо коли руки дійдуть"
  • Ескалація відсутня: якщо makléř не відповів, ніхто про це не дізнається

Яка логіка побудована

  • AI Classifier з Chain of Thought читає кожен лист і визначає category → priority → owner → sla_minutes
  • Reasoning на 3 речення за вимірами naléhavost / obchodní dopad / kontext klienta
  • SLA-таймер: P1 = 15 хв, P2 = 2 год, P3 = 24 год, P4 = 48 год
  • 3-рівнева ескалація: Slack DM власнику → Slack #cx-leads тімліду → email директору
03 Process

Від промпту до робочого агента

01

Prompt design — System Message з 5 блоками

Промпт AI Classifier побудований за фреймворком: Роль (старший координатор підтримки RealityPraha) → Контекст (6 категорій, P1–P4 з SLA-хвилинами, власники) → Інструкція з вимірами (спеціальні тригери — слова "žaloba", "soud", "hrozím" завжди дають P1) → Chain of Thought (обов'язковий аналіз naléhavost / obchodní dopad / kontext klienta) → Формат відповіді (строгий JSON).

role: "Jsi starší koordinátor zákaznické podpory realitní kanceláře RealityPraha..."
trigger: Legal Question → vždy P1, requires_human=true
output: { category, priority, owner, sla_minutes, reasoning, summary, customer_sentiment }
gpt-4o-mini Chain of Thought Structured JSON
02

API integration — OpenAI + Google Sheets + Slack

AI Agent node (n8n LangChain) викликає OpenAI Chat Model і повертає строго структурований JSON. Результат "розплющується" (Flatten Result) у плоский об'єкт із додатковим полем sla_deadline, яке рахується як created_at + sla_minutes у ISO 8601. Дані пишуться в Google Sheets через OAuth2, а сповіщення йдуть у Slack каналу #cx-tickets через окремий webhook на власника.

Gmail OAuth2 OpenAI API Google Sheets API Slack OAuth2
03

n8n workflow — два паралельних трека

Трек входу: Gmail Trigger (1 хв) → Collect Ticket Context → AI Classifier → Flatten Result → фільтр «Не спам?» → паралельно Log → Sheets + Slack: Notify Owner.

SLA Watcher трек: Schedule Trigger (5 хв) → Read Open Tickets → IF (SLA Breached?) → Compute Escalation Level → Switch by Level → L0 (Slack DM owner) / L1 (Slack #cx-leads тімліду) / L2 (email директору).

24 nodes 2 triggers 3-level escalation

05 Screenshots

Як це працює у продакшені

n8n — Track 1 (вхід тікета): Gmail Trigger → Collect Ticket Context → AI Classifier (Chain of Thought) → Flatten Result → Не спам? → паралельно Log → Tickets Sheet + Slack: Notify Owner. У панелі Output — повний JSON класифікації (category, priority, owner, sla_minutes, reasoning, sentiment).
n8n — обидва треки разом: зверху вхідний трек класифікації, знизу SLA Watcher (every 5m) → Read Open Tickets → SLA Breached? → Compute Escalation Level → Switch by Level → Slack: Remind Owner (L0) / Notify TL (L1) / Email Director (L2).
n8n — Track 2 успішний запуск: Switch by Level спрацював, вивід ноди Slack: Remind Owner (L0)1 показує ok: true — повідомлення про прострочення SLA доставлено в Slack.
Tickets Log у Google Sheets: 10 рядків з полями category, priority, owner, sla_minutes, sla_deadline (ISO 8601) та reasoning — 3 речення обґрунтування за naléhavost / obchodní dopad / kontext klienta для кожного тікета.
Slack — нова класифікація: «🚨 Новий тікет — P4. Власник: null, Категорія: Spam, SLA: 2880 хв, Sentiment: neutral».
SLA-ескалація L0: «⚠️ Нагадування — SLA прострочено. Власнику m.simek: тікет P1 прострочено на 65 хв. Тема: Stížnost na makléře — požaduji vrácení zálohy».
