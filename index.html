01 Context

A business that lives in the inbox

RealityPraha is a small real estate office in Prague: 6 people (3 agents, a lawyer, office manager, and director), processing 15–30 inquiries per day through info@realitypraha.cz and the website contact form.

Current process (before automation)

  • All emails land in a shared inbox info@realitypraha.cz
  • Agents manually read and sort mail 2–3 times a day
  • Priority is determined by eye — depending on mood and workload
  • Legal questions and complaints are mixed with standard inquiries and ads

Team and categories

  • Property Inquiry / Viewing Request → Jan Novák
  • Pricing → Petra Horáková
  • Complaint → Martin Šimek (office manager)
  • Legal Question → Eva Čermáková (lawyer)
  • Spam → ignore
02 Problem & Solution

A VIP client waits in the same queue as spam

The biggest leak in customer service is the time between when an email arrives and when someone actually picks it up.

What wasn't working

  • A VIP request to view an 8.5M Kč apartment got lost among ads and walking tips
  • A complaint threatening legal action could wait days for a response
  • No SLA at all — "we'll reply when we get to it"
  • No escalation: if an agent didn't respond, nobody found out

What logic was built

  • AI Classifier with Chain of Thought reads each email and determines category → priority → owner → sla_minutes
  • 3-sentence reasoning by criteria: urgency / business impact / client context
  • SLA timer: P1 = 15 min, P2 = 2 hrs, P3 = 24 hrs, P4 = 48 hrs
  • 3-level escalation: Slack DM to owner → Slack #cx-leads to team lead → email to director
03 Process

From prompt to working agent

01

Prompt design — System Message with 5 blocks

The AI Classifier prompt follows a framework: Role (senior support coordinator at RealityPraha) → Context (6 categories, P1–P4 with SLA minutes, owners) → Criteria instructions (special triggers — words "žaloba", "soud", "hrozím" always yield P1) → Chain of Thought (mandatory analysis: urgency / business impact / client context) → Output format (strict JSON).

role: "Jsi starší koordinátor zákaznické podpory realitní kanceláře RealityPraha..."
trigger: Legal Question → always P1, requires_human=true
output: { category, priority, owner, sla_minutes, reasoning, summary, customer_sentiment }
gpt-4o-mini Chain of Thought Structured JSON
02

API integration — OpenAI + Google Sheets + Slack

The AI Agent node (n8n LangChain) calls OpenAI Chat Model and returns a strictly structured JSON. The result is flattened into a flat object with an additional sla_deadline field, calculated as created_at + sla_minutes in ISO 8601. Data is written to Google Sheets via OAuth2, and notifications are sent to the #cx-tickets Slack channel through a separate webhook to the owner.

Gmail OAuth2 OpenAI API Google Sheets API Slack OAuth2
03

n8n workflow — two parallel tracks

Inbound track: Gmail Trigger (1 min) → Collect Ticket Context → AI Classifier → Flatten Result → "Not spam?" filter → in parallel Log → Sheets + Slack: Notify Owner.

SLA Watcher track: Schedule Trigger (5 min) → Read Open Tickets → IF (SLA breached?) → Compute Escalation Level → Switch by Level → L0 (Slack DM to owner) / L1 (Slack #cx-leads to team lead) / L2 (email to director).

24 nodes 2 triggers 3-level escalation
